Técnicas

Guía rápida de técnicas de prompting

Antes de explorar el menú, aquí tienes un resumen práctico de cada técnica. Ábrela para ver cuándo usarla, su objetivo y un mini-ejemplo.

ASPECT — Estructura completa del prompt

Cuándo usar: cuando necesitas instrucciones claras, completas y reutilizables.

Objetivo: no olvidar elementos clave: Acción, Sujeto, Propósito, Ejemplos, Restricciones, Tono.

### ACCIÓN Resume · ### SUJETO Informe Q1 · ### PROPÓSITO Directivos
    ### EJEMPLO "Ingresos ↑12%" · ### RESTRICCIONES ≤120 palabras, viñetas · ### TONO Ejecutivo
Chain-of-Thought (CoT) — Razonamiento paso a paso

Cuándo usar: tareas lógicas, matemáticas o con múltiples pasos.

Objetivo: pedir al modelo que explique su proceso antes de dar la respuesta.

### INSTRUCCIÓN
    Razona paso a paso y luego da la respuesta final.
Prompt Layering — Capas secuenciales

Cuándo usar: problemas complejos que conviene dividir (extraer → clasificar → resumir).

Objetivo: controlar mejor cada sub-tarea y depurar con facilidad.

CAPA 1: extraer entidades → CAPA 2: sentimiento → CAPA 3: informe
Instructive Prompting — Respuestas estrictas y estructuradas

Cuándo usar: cuando el resultado alimenta otro sistema (CSV/JSON/YAML).

Objetivo: formato y reglas claras para automatización.

{ "task":"classify", "labels":["positive","negative"], "format":"csv" }
Iterative Prompt Refinement (IPR) — Mejorar por ciclos

Cuándo usar: si el primer resultado no cumple calidad: ajustar → reintentar.

Objetivo: converger hacia precisión, relevancia y costo eficiente.

V1 → evaluar → ajustar restricciones/tono/ejemplos → V2 → ...
Zero-shot — Sin ejemplos

Instrucción precisa sin ejemplos previos. Rápido, pero sensible a ambigüedad.

Few-shot — Con pocos ejemplos

Guía la salida con 2–5 ejemplos representativos. Mejora consistencia.

Role-based — Asignar un rol

“Actúa como…” para activar conocimientos, estilos y restricciones de un perfil.

Meta-prompting — Que el modelo diseñe su plan

Pide que proponga su propio prompt/plan antes de responder. Útil para tareas abiertas.

Prompt Chaining — Encadenar prompts

Salida A → Prompt B. Ideal para pipelines (limpieza → análisis → reporte).

Cuando elijas una técnica en el menú lateral, verás ejemplos ampliados, variantes y buenas prácticas.
